# vNext Phase 1 Closeout

This document marks the end of the first promoted `vNext` delivery wave on `codex/vnext-integration`.

## Scope Closed In Phase 1

- worktree + lane + integration delivery model
- capability registry, scorecard, and maturity-aware doctor
- first-pass cloud surface productization across `ResponseAPI`, file/qfile workflow, batch, admin, censor, account, log, OCR, ASR, TTS, image, and video
- first-pass runtime productization across `SessionStore`, replay/restore helpers, thread lifecycle, and resumable thread helpers
- first-pass node productization across MCP host/transport probing, structured resource/prompt reads, and live-verify hooks
- repository-side validation chain:
  - package/docs/template smoke
  - live verification gate
  - live verification JSON / markdown artifacts
  - aggregated verification report artifact

## Phase 1 Exit Criteria

Status: met on `codex/vnext-integration`

- integration branch is clean after fresh verification
- `npm run prepublishOnly` passes
- capability scorecard and live-verification artifacts are generated by tracked tooling
- live verification has a first-class CLI/CI entrypoint
- phase scope is documented in tracked workflow + durable plans

## Known Deferred Work

- live verification is still credential-gated rather than an unconditional PR hard gate
- capability truth is still partly manual; it is not yet fully synced from docs + live evidence
- runtime product story is improved but not complete
- MCP / node integrations have stronger unit coverage than real interop evidence
- several beta / experimental modules still lack promotion-ready evidence

## Merge Readiness

- Phase 1 is ready to be treated as a coherent merge candidate.
- Direct mutation of the root `main` worktree was intentionally avoided because the user's checked-out `main` workspace contains unrelated local changes.
- Further landing onto `main` should happen from a clean main checkout or a dedicated landing worktree.
